# Architecture Documentation

This repository contains the architecture documentation for all repositories in the project.

## Repositories
- **Frontend Repositories**:
  - `DDV-DAMAYAN-FE`: Frontend for the Disaster DAMAYAN Management System
  - `DDV-BayaniHub-FE`: Frontend for the Disaster BayaniHub Management System
  - `DDV-HopeCard-FE`: Frontend for the Disaster HopeCard Management System
  - `SC-FE`: Frontend for the Supply Chain Management System
  - `LAP-pakiSHIP-FE`: Frontend for the Logistics and Parking pakiSHIP Management System
  - `LAP-pakiPARK-FE`: Frontend for the Logistics and Parking pakiPARK System
  - `HR-Blues-Clues-FE`: Frontend for the Human Resources Blues CLues Management System
  - `SID-servEASE-FE`: Frontend for the Service in Demand Management System
  - `CO-FE`: Frontend for the Campus One Management System
- **Mobile Repositories**:
  - `DDV-Mobile`: Mobile app for Disaster Management
  - `SC-Mobile`: Mobile app for Supply Chain Management
  - `LAP-Mobile`: Mobile app for Logistics and Parking Management
  - `HR-Mobile`: Mobile app for Human Resources Management
  - `SID-Mobile`: Mobile app for Service in Demand Management
  - `CO-Mobile`: Mobile app for Campus One Management
- **Backend Repositories**:
  - `DDV-BE`: Backend for Disaster Management
  - `SC-BE`: Backend for Supply Chain Management
  - `LAP-BE`: Backend for Logistics and Parking Management
  - `HR-BE`: Backend for Human Resources Management
  - `SID-BE`: Backend for Service in Demand Management
  - `ApiCenter-BE`: Backend for Api Center Management
  - `CO-BE`: Backend for CampuOne Management

## Purpose
This documentation is intended to provide an overview of the repository structure, purpose, and how each part contributes to the overall system.
